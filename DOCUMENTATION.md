# Naomi — документация (v0.2.0)

Naomi — это локальный чат-компаньон: живая, тёплая собеседница со своим характером. Просто болтает. Работает на **твоей подписке ChatGPT**, без платного API.

## Как это работает

Когда ты входишь через ChatGPT, токен подписки сохраняется в `~/.codex/auth.json`. Бэкенд Наоми берёт этот токен и шлёт запрос **напрямую** в бэкенд OpenAI (`chatgpt.com/backend-api/codex/responses`), подставляя **свой** системный промпт — личность Наоми. Поэтому она отвечает как Наоми, а не как «кодинг-агент».

Это «путь B»: не через программу Codex, а собственным клиентом. Системный промпт под нашим контролем — отсюда чистая личность и экономный запрос (~100 входных токенов вместо ~16 000 у Codex CLI).

## Архитектура

| Часть | Файл | Назначение |
|---|---|---|
| Личность | `naomi.md` | характер Наоми (читается на каждый запрос) |
| Правила | `pravila.md` | как общаться и как не общаться (заготовка) |
| Сервер | `backend/server.py` | API + отдаёт фронт |
| Клиент LLM | `backend/oai.py` | запрос в responses-endpoint + обновление токена |
| Вход | `backend/auth.py` | «Sign in with ChatGPT» (PKCE OAuth) |
| Телеграм | `backend/telegram.py` | мост в Telegram (токен в `data/telegram.json`, вне git) |
| Интерфейс | `frontend/` | React без сборщика (esbuild) |

Один процесс (FastAPI + uvicorn) отдаёт и статику фронта, и `/api/*` — поэтому всё работает с одного origin, без CORS.

## API

| Эндпоинт | Метод | Что делает |
|---|---|---|
| `/api/chat` | POST | сообщение → ответ Наоми `{reply, thought, usage}` |
| `/api/history` | GET | видимая история (для восстановления) |
| `/api/settings` | GET/POST | модель и уровень размышления |
| `/api/auth/status` | GET | статус подписки (план, залогинен ли) |
| `/api/auth/login` | POST | запуск входа через ChatGPT |
| `/api/docs` | GET | этот документ |
| `/api/health` | GET | версия и состояние |
| `/api/events` | GET (SSE) | keepalive-канал |

Токен **никогда** не отдаётся фронту — `/api/auth/status` сообщает лишь факт логина и план.

## Настройки (⚙️ → Агент)

**Модель:**

| ID | Описание |
|---|---|
| `gpt-5.5` | самая умная (по умолчанию) |
| `gpt-5.4` | сбалансированная |
| `gpt-5.4-mini` | самая быстрая и лёгкая |

**Размышление:** `low` (по умолчанию) · `medium` · `high` · `xhigh`. Низкий — быстро и естественно для болтовни; выше — думает дольше.

Список моделей зависит от твоей подписки (источник истины — `codex debug models`).

## Запуск и разработка

```sh
./run.sh        # → http://127.0.0.1:8765
```

Первый вход: ⚙️ → **Аккаунт** → «Войти через ChatGPT».

Личность правится в `naomi.md` / `pravila.md` — изменения применяются **сразу**, без перезапуска.

Фронтенд собирается из `.jsx` в `.js` через `frontend/build.sh` (esbuild). После правок `app.jsx` / `ui-kit.jsx` нужно пересобрать.

## Telegram (опционально)

Чтобы говорить с Наоми в Telegram: создай бота у [@BotFather](https://t.me/BotFather), узнай свой числовой chat id и положи `backend/data/telegram.json`:

```json
{ "token": "<токен бота>", "chat_id": "<твой id>" }
```

Файл в `.gitignore` — токен в репозиторий не попадёт. Мост поднимется при старте сервера; Наоми отвечает **только владельцу** (твоему chat id), той же личностью, что и в вебе.

## Приватность

- Токен подписки живёт в `~/.codex/auth.json` (вне репозитория) и не попадает в git.
- История чата (`backend/data/`) и локальный конфиг (`.claude/launch.json`) — в `.gitignore`.
- В коммитах используется noreply-почта, личный email не светится.

## Известные ограничения

- Подключение через подписку — «серая зона» по отношению к ToS OpenAI; используется осознанно для личного некоммерческого компаньона.
- Ответ приходит целиком (фронт сам анимирует его по словам), пословный стриминг с сервера — в планах.
