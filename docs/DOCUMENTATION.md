# Naomi — документация (v0.9.0)

Naomi — локальный чат-компаньон: живая, тёплая собеседница со своим характером. Просто болтает, ищет в интернете когда нужно, знает реальные время и состояние «умного дома». Работает на **твоей подписке ChatGPT**, без платного API.

## Как это работает

Когда ты входишь через ChatGPT, токен подписки сохраняется в `~/.codex/auth.json`. Бэкенд берёт этот токен и шлёт запрос **напрямую** в бэкенд OpenAI (`chatgpt.com/backend-api/codex/responses`), подставляя **свой** системный промпт — личность Наоми. Поэтому она отвечает как Наоми, а не как «кодинг-агент».

Это «путь B»: не через программу Codex, а собственным клиентом. Системный промпт под нашим контролем — отсюда чистая личность и экономный запрос (~1700 токенов личности вместо ~16 000 у Codex CLI). Токен **никогда** не отдаётся фронту.

## Чат и контекст

Веб и Telegram — **один общий тред**: контекст копится и виден в обоих местах, Наоми помнит разговор независимо от того, где ты пишешь. Тред живёт **в памяти сервера** → **сбрасывается при перезапуске** агента (рестарт = чистый лист); кнопка-корзина в шапке очищает его вручную.

- Пишешь в **Telegram** → сообщение и ответ **сразу появляются в вебе** (live, через SSE).
- Пишешь в **вебе** → в Telegram тихо, но контекст общий.

Ответ в вебе идёт **реальным стримом** (SSE): слова появляются по мере генерации, каждое всплывает анимацией. В Telegram — одним готовым сообщением (у платформы нет токен-стрима; цельный ответ читается чище).

### Живой контекст «[Сейчас] …»

Перед последней репликой в запрос **эфемерно** (не сохраняясь в историю, в самом конце — кеш личности цел) подкладывается блок с реальными датой/временем (с поясом) и снимком умного дома. ~110 токенов. Поэтому Наоми знает «сейчас», кто дома, погоду — и отвечает по факту, а не гадает.

## Структура проекта

```
backend/      Python (FastAPI): API + отдаёт фронт
frontend/     React без сборщика (esbuild), панель «умный дом»
home/         модульный слой «умный дом» (датчики + контекст)
persona/      личность: naomi.md + pravila.md
docs/         эта документация
README.md  LICENSE  run.sh  .gitignore
```

| Часть | Файл | Назначение |
|---|---|---|
| Личность | `persona/naomi.md` | характер Наоми (читается на каждый запрос) |
| Правила | `persona/pravila.md` | как общаться и как не общаться |
| Дом (персона) | `home/persona.md` | как Наоми держится про умный дом |
| Сервер | `backend/server.py` | API + отдаёт фронт, единый тред, инъекция контекста |
| Клиент LLM | `backend/oai.py` | запрос в responses-endpoint, стрим, тул `web_search`, обновление токена |
| Веб-поиск | `backend/search.py` | Tavily (ключ в `data/services.json`, вне git) |
| Вход | `backend/auth.py` | «Sign in with ChatGPT» (PKCE OAuth) |
| Телеграм | `backend/telegram.py` | мост (токен в `data/telegram.json`, вне git), markdown→MarkdownV2 |
| Умный дом | `home/` | оркестратор + модули `presence/toilet/climate/indoor/weather/vacuum` |
| Интерфейс | `frontend/app.jsx` | весь UI (чат, настройки, панель дома) |

Один процесс (FastAPI + uvicorn) отдаёт и статику фронта, и `/api/*` — всё с одного origin, без CORS. Сервер слушает только `127.0.0.1` (локальный доступ).

## Умный дом

Слой `home/` — модульный. Каждый «прибор» — папка-модуль в `home/modules/` со своим кодом и единым контрактом (`KEY` / `DEFAULT` / `context()` [/ `persona()`]). Оркестратор (`home/__init__.py`) собирает со всех модулей состояние, контекст-строку и персону, маршрутизирует изменения.

Сейчас датчики **фейковые** — состояние задаёт владелец кликами в панели (кнопка-домик в шапке, выезжает справа): люди (дома/комната), кондиционер, туалет, погода за окном, температуры по комнатам, пылесос. Состояние — `home/data/state.json` (вне git). Наоми видит изменения **на следующем же сообщении**.

Добавить реальный модуль: папка в `home/modules/`, импорт в `home/registry.py`; свои зависимости → его `requirements.txt`, секреты → его `config.json` (вне git); позже `tools()` для управления и `async start()` для живого подключения.

## Веб-поиск (Tavily)

Если в ⚙️ → **Аккаунт** добавлен ключ Tavily, у Наоми появляется инструмент `web_search`: она **сама** решает, когда искать (бережём лимит), формулирует запрос **на английском**, и отвечает по результатам. В вебе во время поиска видно строку «ищу в интернете…». Ключ хранится в `backend/data/services.json` (вне git), наружу не отдаётся.

## API

| Эндпоинт | Метод | Что делает |
|---|---|---|
| `/api/chat` | POST | сообщение → **SSE-стрим** токенов (события `data: {"t":"delta"\|"tool"\|"done"\|"error", …}`; у `delta` — поле `d`, у `tool` — `q`) |
| `/api/history` | GET | видимый тред (для восстановления) |
| `/api/reset` | POST | очистить общий тред |
| `/api/home` | GET / POST | состояние умного дома / патч (клик в панели) |
| `/api/services/tavily` | GET / POST | статус / сохранить ключ Tavily (ключ наружу не отдаётся) |
| `/api/settings` | GET / POST | модель и уровень размышления |
| `/api/auth/status` | GET | статус подписки (план, залогинен ли) |
| `/api/auth/login` | POST | запуск входа через ChatGPT |
| `/api/health` | GET | версия и состояние |
| `/api/events` | GET (SSE) | keepalive + live-зеркало Telegram→веб |

## Настройки (⚙️)

Три вкладки: **Агент** (модель + размышление), **Аккаунт** (вход ChatGPT + ключ Tavily), **О Наоми**.

**Модель:** `gpt-5.5` (умнейшая, по умолчанию) · `gpt-5.4` (сбаланс.) · `gpt-5.4-mini` (быстрая). Список зависит от подписки (`codex debug models`).

**Размышление:** `low` (по умолчанию) · `medium` · `high` · `xhigh`. Низкий — быстро и естественно для болтовни; выше — думает дольше.

## Запуск и разработка

```sh
./run.sh        # → http://127.0.0.1:8765
```

Первый вход: ⚙️ → **Аккаунт** → «Войти через ChatGPT».

Личность правится в `persona/naomi.md` / `persona/pravila.md` (и `home/persona.md`) — изменения применяются **сразу**, без перезапуска.

Фронтенд собирается из `.jsx` в `.js` через `frontend/build.sh` (esbuild). После правок `app.jsx` / `ui-kit.jsx` нужно пересобрать.

Зависимости бэкенда — `backend/requirements.txt` (fastapi, uvicorn, httpx, telegramify-markdown). Тесты — `tests/` (`python -m pytest` или `./tests/run.sh`).

## Автозапуск (macOS)

LaunchAgent `~/Library/LaunchAgents/com.naomi2.server.plist` поднимает сервер (и Telegram-мост) при входе и перезапускает при падении (`backend/.venv/bin/python -m uvicorn server:app --app-dir backend --port 8765`, `RunAtLoad` + `KeepAlive`).

- загрузить: `launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.naomi2.server.plist`
- перезапуск (= сброс контекста): `launchctl kickstart -k gui/$(id -u)/com.naomi2.server`
- выключить автозапуск: `launchctl bootout gui/$(id -u)/com.naomi2.server`

⚠️ Если автозапуск включён — **не** запускай `./run.sh` отдельно: два сервера на :8765 конфликтуют (и два Telegram-поллера → ошибка 409).

## Telegram (опционально)

Создай бота у [@BotFather](https://t.me/BotFather), узнай свой числовой chat id, положи `backend/data/telegram.json`:

```json
{ "token": "<токен бота>", "chat_id": "<твой id>" }
```

Файл в `.gitignore`. Мост поднимется при старте; Наоми отвечает **только владельцу**, той же личностью, что и в вебе. Ответы форматируются (жирный/списки/код) через MarkdownV2; если разметка не зашла — фолбэк на обычный текст.

## Приватность и безопасность

- Токен подписки — в `~/.codex/auth.json` (вне репозитория, права `0600`).
- Состояние/секреты — в `backend/data/`, `home/data/`, `home/**/config.json`, `**/telegram.json` — всё в `.gitignore`, в репозиторий не попадает.
- Сервер слушает только `127.0.0.1`; API без аутентификации — граница безопасности это локальный доступ (однопользовательское приложение).
- В коммитах — noreply-почта, личный email не светится.

## Известные ограничения

- Подключение через подписку — «серая зона» по отношению к ToS OpenAI; используется осознанно для личного некоммерческого компаньона.
- Датчики умного дома пока фейковые (задаются кликами); реальные устройства — на будущее.
